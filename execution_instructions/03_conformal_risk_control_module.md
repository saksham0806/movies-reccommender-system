# Module 3: Conformal Risk Control (CRC) Layer

## Overview
The Conformal Risk Control (CRC) module asynchronously attaches to the Denoising Transformer inference stage. Its purpose is to function as a dynamic guardrail, calculating risk thresholds during runtime to prevent the recommendation of unsafe or offensive content without needing to retrain the underlying model (De Toni et al., 2025).

## 1. Safety Vector Space Setup
- Using the MovieLens `movies.dat` metadata, map specific genres or tags to risk profiles. For example, define the "Horror" and "Crime" genres as the "risky/harmful" vector space for testing guardrails.
- Pre-select a pool of globally "safe", family-friendly content (e.g., movies containing the "Animation", "Comedy", or "Children's" genre tags) to act as fallback recommendations.

## 2. Kernel Density Estimator Function ($p_h$)
- Implement the $p_h$ function using Kernel Density Estimation (KDE) or an equivalent probabilistic approach.
- **Input**: The recommendation candidate item target $Y_i$.
- **Output**: The likelihood probability that the item falls within the unwanted/risky feature vector space.

## 3. Risk Calculation ($R_i$)
- Calculate the residual probability risk function $R_i$ based on $p_h(Y_i)$.
- Define a strict safety threshold ($\tau$). The smaller the $R_i$, the safer the video. 

## 4. Dynamic Filtering Execution
Inject the CRC layer straight into the inference pipeline:
1. For every suggestion $Y_i$ generated by Module 2, evaluate $R_i$.
2. **Conditional Check**: `if R_i > tau:`
3. **Failsafe Swap**: Swap $Y_i$ with an item from the pre-selected safe content pool.
4. Yield the final, filtered list of recommendations.

This module guarantees that harmful anomalies do not permeate the final list served to the end-users.
